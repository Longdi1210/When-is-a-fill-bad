# Implementation Note: Repository Inspection

## Current State

The repository is a compact synthetic limit-order-book prototype. It contains:

- `src/lob_sim/book.py`: reusable price-time-priority matching engine with limit, market, and cancel orders.
- `src/lob_sim/agents.py`: synthetic heterogeneous agents that generate noise, informed, resilience, and cancellation order flow.
- `src/lob_sim/simulation.py`: deterministic simulation loop that writes snapshots, trades, and features.
- `src/lob_sim/features.py`: reusable microstructure features such as queue imbalance, microprice deviation, spread bps, and signed trade flow.
- `scripts/run_demo.py`, `scripts/run_research_sweep.py`, `scripts/run_thesis_transfer_experiment.py`: existing synthetic experiments.
- `tests/`: basic tests for matching and feature signs.
- `outputs/`: generated synthetic snapshots/trades/features and previous thesis-transfer portfolio summaries.

## Reusable Work

- The matching engine is useful for generating synthetic event streams and documenting price-time priority assumptions.
- Existing book snapshots, trades, and microstructure features are useful raw inputs for hypothetical passive-order replay.
- The synthetic agent setup already embeds market stress through informed order flow, cancellation pressure, resilience liquidity supply, and order-flow autocorrelation.
- Existing tests can be preserved and extended.

## Duplicated Or Incomplete Components

- `run_research_sweep.py` and `run_thesis_transfer_experiment.py` duplicate scenario-sweep logic, but they are not directly relevant to the new research question.
- There is no hypothetical passive-order construction.
- There is no queue-ahead model, no fill/censoring labels, no post-fill markout labels, and no expected posting value.
- There are no chronological train/validation/test splits.
- There are no fill-probability or markout models.
- Existing outputs show spread/imbalance/volatility/resilience, but not fill probability, toxicity, or economic value.

## Embedded Assumptions

- Data are synthetic, generated by a simple agent-based LOB model.
- Price-time priority is assumed by the matching engine.
- Existing snapshots contain top-of-book and five-level depth, not full market-by-order history.
- Existing trades include aggressor side and quantity, so trade-driven depletion can be measured at the top of book.
- Cancellations are not currently emitted as explicit events in output files.
- Queue ahead for hypothetical passive orders cannot be reconstructed exactly from committed output alone; it must be proxied from displayed same-side top-of-book depth.

## Existing Fill And Adverse-Selection Results

No current result estimates passive-fill probability, adverse post-fill markout, or posting value. Existing sweep results only show that synthetic information shocks widen spread and increase microprice deviation.

## Design Direction

Build on the existing synthetic generator and features, but refocus the project around one narrow question:

When does a higher passive-order fill probability make the order economically worse because the fill is more adversely selected?

The new pipeline should generate synthetic events, construct hypothetical buy/sell passive orders at the touch, label fills and signed markouts with no look-ahead features, fit interpretable baseline models, compare fill-only ranking with value-based ranking, and save figures/tables reproducibly.
